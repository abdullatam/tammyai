# Contributing to Tammy AI

We are building the future of emotional intelligence infrastructure. We welcome contributors who care deeply about psychology, performance, and beautiful code.

## Guiding Principles
1. **Preserve the Soul:** Tammy should feel warm, sharp, and deeply empathetic. Do not submit PRs that make her sound like a generic corporate bot.
2. **Speed is a Feature:** The AI orchestration layer must remain blazing fast. Avoid blocking the FastAPI event loop.
3. **Privacy First:** Never log raw user messages or memory vectors in plaintext.

## Development Workflow
1. Branch off `develop`.
2. Prefix your branches: `feature/`, `bugfix/`, `docs/`, `refactor/`.
3. Ensure the test suite passes (`pytest tests/`).
4. Submit a Pull Request targeting `develop`.

## Code Style
- Python: `black` and `flake8` compliance.
- JS/React: Functional components, React Hooks, and standard ESLint.
